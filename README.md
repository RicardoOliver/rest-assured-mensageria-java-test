## EstratГ©gia de Testes

### Por que Awaitility?

Sistemas assГӯncronos nГЈo tГӘm um delay de propagaГ§ГЈo fixo. `Thread.sleep()` gera testes frГЎgeis que esperam tempo demais ou de menos. O Awaitility realiza polling com intervalos e timeouts configurГЎveis, assertando *quando* o estado estГЎ pronto вҖ” e nГЈo *apГіs* um delay arbitrГЎrio.

```java

    .pollInterval(500, MILLISECONDS)
    .untilAsserted(() ->
        given()
            .get("/messages/{id}", messageId)
        .then()
            .statusCode(200)
    );
```

### CenГЎrios de Teste

| CenГЎrio | Resultado Esperado |
|---------|-------------------|
| Mensagem vГЎlida, primeira entrega | Persistida вҶ’ `GET 200` |
| `messageId` duplicado | Descartada silenciosamente, idempotente вҶ’ `GET 200` (original) |
| Payload invГЎlido (regra de negГіcio) | Roteada para DLQ вҶ’ `GET 404` |
| Consumidor indisponГӯvel e depois recuperado | Mensagem enfileirada, processada na recuperaГ§ГЈo вҶ’ `GET 200` |
| Broker inacessГӯvel no momento da publicaГ§ГЈo | API retorna `5xx`, nenhum estado parcial |
